Working Locally With Docker

Question: Wait, what? If Im just working locally with a static website, then why oh why would I want to use Docker?

Answer 1: I've found that there are many cases where JavaScript doesn't run correctly if you're not running the code on top of a web server.

Answer 2: If the intention of this website is to provide you with up-to-date tools for web development, I want to have all tutorials provide a good strong foundation. Therefore, I feel strongly that Docker should be incorporated into every developers toolbelt, especially those who handle system admistration or oversee any code deployment.
